## 提交前固定检查

每次改动前：

- [ ] 执行 `git status --short --branch`
- [ ] 确认没有要保护的用户未提交改动

每次提交前：

- [ ] `node --check frontend/src/main.js`
- [ ] `node --check shared/game.js`
- [ ] `node --check shared/config.js`
- [ ] `git diff --check`
- [ ] `git status --short --branch`

## AI 应用记录

本项目大量使用 AI 协作：

- Codex：代码实现、调试、文档维护、Git 操作、部署同步
- ChatGPT Pro：玩法讨论、版本规划、提交材料润色
- AIArt：主界面、背景、标题、美术素材生成
- Game Studio 插件：UI 风格和高品质游戏视觉方向辅助

经验：

- AI 很擅长快速搭原型，但规则要写清楚。
- 三消细节不能只看表象，要明确掉落来源、木箱阻挡、无效交换、特殊棋子身份等规则。游泳模式当前不要启用斜向掉落。
- 每轮稳定后要及时提交、更新文档，避免新对话失去上下文。
